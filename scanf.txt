绝大多数输入输出流完全缓存，数据流先进入缓冲区，写满时才会刷新，但标准输入输出流不必写满。
scanf暂存于缓冲区，敲回车时开始工作，scanf读的为缓冲区的数据，缓冲区为空时才等待用户输入。
若不匹配格式参数要求，将会随机生成。若格式说明参数中有空格,将自动被忽略。
scanf引起的死循环，它要跳过所有非数字的字符专接收数字，那么输入了字符它就一直留在输入缓冲区；只要缓冲区有字符，scanf就要去读，一读不是数字，它就跳过，试图等到输入缓冲区没有字符了才等待你输入……如此反复。
1.fflush(stdin)语句清除缓冲区多余字符；
2.getchar()读取缓冲区字符
int clearstdin() {//清缓存
	int c;
	while ((c = getchar()) != '\n' && c != EOF);
	return 0;
}
